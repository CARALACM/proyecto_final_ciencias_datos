# INSTRUCCIONES PROYECTO FINAL CIENCIAS DE DATOS

## Definición del problema.

1. Buscar base de datos. Debe tener al menos 100000 instancias.
2. ¿Qué pregunta quiero resolver? Identificar los objetivos del proyecto.
3. Entender el contexto. ¿A qué se debe la pregunta?
4. Evaluar los recursos disponibles. Se puede obtar por usar WebScrapping, APIs.
5. Establecer el alcance del proyecto. Normalmente un proyecto profesional lleva aproximadamente 9 meses (consideraremos 9 meses de paga).

## Recopilación de datos.

### Piramide de datos:
1. Identificar fuentes. Reconocer las diferentes fuentes de datos relevantes para el proyecto.
2. Extracción. Recolectar los datos de manera eficiente y estructura.
3. Validación. Verificar la calidad y coherentes de los datos recopilados.

Para saber si tenemos fiabilidad en podemos revisar la tabla Fischer-Colton.

## Modelado.

1. Selección de técnicas analíticas.
2. Selección del modelo.
3. Entrenamiento del modelo.
4. Ajuste de hiperparámetros.
5. Evaluación del modelo.
6. Análisis de errores.
7. Optimización del modelo.
8. Comparación entre modelos.

### Seleccionar las técnicas.

Dependiendo de las preguntas a resolver y sobre todo nuestros datos debemos escoger una técnica para la preducción, se puede obtar por:

* Árboles de decisión. 

* Agrupamiento
    - K-Means.
    - Agrupamiento jerárquico.
    - DBSCAN.

* Clasificación
    
* Redes neuronales. 
 - Perceptrón.
 - Perceptrón multicapa.
 
* Redes neuronales profundas.

* Redes neuronales convolucionales.

Se puede utilizar regresiones lineales, pero cuidando que no se genere demasiado error.

### Validación de resultados.

Se debe buscar la validación estadística de los resultados con p-student, p-value, etc.

Se debe hacer un analísis de sensibilidad, alterar un poco los valores para ver si llegamos al mismo resultado.

## Interpretación y comunicación.

Esto se piensa para entregar a quién te contrató para el proyecto.

* Interpretación de resultados.
* Contextualización de los resultados.
* Identificación de insights.
* Preparación de informes y presentaciones.
* Propuesta de acciones recomendadas.
* Seguimiento y revisión. 

Dependiendo de la audiencia se pedirá más o menos nivel de detalle matemático.
